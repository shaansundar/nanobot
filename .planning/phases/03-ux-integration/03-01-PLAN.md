    **Step 3 -- _build_command (per D-04):**
    Add `system_prompt: str | None = None` parameter to `_build_command()`. When provided, append `["--append-system-prompt", system_prompt]` to the command list BEFORE the prompt argument. This preserves Claude Code's built-in system prompt while adding the persona (per D-01: use --append-system-prompt, NOT --system-prompt).

    **Step 4 -- chat() persona extraction (per D-03):**
    In `chat()`, after extracting the prompt, if `self._persona_passthrough` is True, find the first message with `role == "system"` and extract its `content` string. Pass it as `system_prompt` to `_build_command()`. If no system message found or persona_passthrough is False, pass None.

    **Step 5 -- _parse_result verbosity filtering (per D-06, D-07, D-08, D-09):**
    After extracting `result_text` in `_parse_result()`, apply verbosity filtering BEFORE constructing the LLMResponse:
    - `"full"` (D-06): No filtering, use result_text as-is.
    - `"final"` (D-07): Call `self._extract_final_response(result_text)` -- split text by double newlines, identify and remove lines that match tool action patterns (lines starting with common action verbs: "Read", "Wrote", "Edited", "Searched", "Ran", "Created", "Deleted", "Listed", "Viewed" followed by file paths or descriptions). Return the remaining text (the final response paragraphs).
    - `"summarized"` (D-08): Call `self._summarize_actions(result_text)` -- identify tool action lines using the same verb patterns, condense each to a one-line summary (e.g., "Edited src/main.py"), then append the final response text after a blank line separator.

    Add two private helper methods:
    - `_extract_final_response(self, text: str) -> str`
    - `_summarize_actions(self, text: str) -> str`

    Use `import re` at the module level. The action verb pattern: `r"^(Read|Wrote|Edited|Searched|Ran|Created|Deleted|Listed|Viewed|Updated|Moved|Copied)\b"` matched against each paragraph (split by `\n\n`).
